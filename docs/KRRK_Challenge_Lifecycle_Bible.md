# Challenge 철학
*Challenge는 KRRK의 핵심이다. 게임은 수단이고, Challenge가 본질이다.*

사용자는 게임을 하러 오는 게 아니다. '이번주 독박왕이 누구냐', '오늘 커피 누가 사냐'를 결정하러 온다.
Challenge는 그 결정을 만들어내는 판이다.

# Challenge 구성 항목
누구나 챌린지를 제안할 수 있다. 제안 즉시 시작된다. 항목은 4가지다.

## 1. 기간

| 기간 | 설명 | 시즌 누적 |
| --- | --- | --- |
| 단판 | 지금 이 자리에서 한 번. 게임 끝나면 챌린지 종료. | 없음 |
| 1주 | 7일간 매일 한 판씩. 주말 독박왕 공개. | 있음 |
| 1달 | 한 달간 매일 한 판씩. 월말 독박왕 공개. | 있음 |

단판은 즉흥 게임용. 1주/1달은 시즌 누적용.

## 2. 게임

| 게임 | 핵심 감정 | 플레이 시간 |
| --- | --- | --- |
| 구슬 레이스 | 역전, 추월, 탈락 | 20~30초 |
| 폭탄 게임 | '제발 나 아니길' | 10~20초 |
| 타이머 게임 | 손 떨림, 심리전 | 30~60초 |

모든 게임은 10초 안에 이해, 30~60초 안에 종료. 결과는 캡처해서 단톡방 공유 가능.

**폭탄 게임은 꼴찌 정하기 전용 — 1등 베네핏 챌린지에서는 선택 불가.**

## 3. 벌칙 / 보상

| 유형 | 설명 |
| --- | --- |
| 꼴찌 벌칙 | 꼴찌가 내용을 이행. 예: '꼴찌가 술값', '꼴찌가 운전 담당' |
| 1등 베네핏 | 1등이 혜택을 받음. 예: '1등은 커피 면제', '1등은 다음 모임 장소 선택' |

내용은 자유 텍스트로 입력. 벌칙과 보상은 심리적으로 완전히 다른 게임이다.

## 4. 공개 방식

| 방식 | 설명 |
| --- | --- |
| 실시간 공개 | 게임 결과가 즉시 모든 멤버에게 공개됨. |
| 종료 후 공개 | 챌린지 기간 동안 순위 비공개. 종료 시점에 한 번에 공개. |

기본값: 종료 후 공개. 히든 랭킹이 긴장감을 유지한다.

# Challenge 라이프사이클
## STEP 1 — 챌린지 제안
그룹 내 누구나 챌린지를 제안할 수 있다.

**플로우**
1. 그룹 홈에서 '챌린지 제안' 탭
2. 기간 선택 (단판 / 1주 / 1달)
3. 게임 선택
4. 벌칙 유형 선택 + 내용 입력
5. 공개 방식 선택
6. 제안 즉시 챌린지 시작

**핵심 원칙**
- 수락 절차 없음. 제안 = 시작.
- 동시에 여러 챌린지 운영 가능.
- 그룹 전체 자동 참여.

## STEP 2 — 게임 참여
챌린지가 시작되면 그룹 멤버 전체가 자동으로 참여 상태가 된다.

**단판 챌린지**
- 그 자리에서 즉시 게임 실행.
- 게임 종료 = 챌린지 종료.

**1주 / 1달 챌린지**
- 매일 게임 알림 발송.
- 당일 미참여 시 자동 패배 처리.
- 결과는 매일 누적.

*미참여 = 자동 패배. 단순하게 간다. 친구들끼리니까.*

## STEP 3 — 히든 랭킹
종료 후 공개 방식의 경우, 챌린지 기간 동안 순위는 공개되지 않는다.

**멤버가 볼 수 있는 것**
- 자신의 현재 상태만 확인 가능.

| 상태 | 의미 |
| --- | --- |
| 안전권 | 현재 상위권. 여유 있음. |
| 위험권 | 하위권 근접. 조심 필요. |
| 매우 위험 | 현재 꼴찌권. 독박 위기. |

구체적인 순위와 점수는 공개되지 않는다. 긴장감 유지가 목적이다.

## STEP 4 — 챌린지 종료

**단판**
- 게임 종료 즉시 결과 공개.
- 패배자 확정 → 벌칙/보상 실행.

**1주 / 1달**
- 기간 종료 시점에 결과 계산.
- 순위 공개 — 1위 / 2위 / 3위 / 독박왕.
- 결과 카드 생성 → 단톡방 공유 가능.
- 업적 반영 (독박왕, 연승왕, 역전왕 등).

*시즌 공개는 이벤트여야 한다. 드라마틱하게.*

## STEP 5 — 다음 시즌
챌린지 종료 후 같은 그룹이 새 챌린지를 시작하는 것이 핵심 루프다.

**북극성 지표**
**시즌 종료 후 같은 그룹이 다음 챌린지를 시작하는 비율.**

# Challenge State Machine

| ACTIVE | 챌린지 진행 중. 게임 참여 가능. 히든 랭킹 운영. |
| --- | --- |

| FINISHED | 기간 종료. 결과 계산 중. |
| --- | --- |

| REVEALED | 결과 공개 완료. 독박왕 확정. 카드 공유 가능. |
| --- | --- |

| ARCHIVED | 기록 보존. 조회만 가능. |
| --- | --- |

# 시나리오 예시
## 시나리오 A — 단판 (술자리 즉흥)
민수, 지훈, 현우, 태양이 술집에서 만났다.

1. 현우가 앱 켜서 챌린지 제안.
2. 기간: 단판 / 게임: 폭탄 게임 / 벌칙: 꼴찌가 다음 안주 / 공개: 실시간.
3. 제안 즉시 시작. 그룹 전체 자동 참여.
4. 폭탄 게임 30초. 지훈 폭탄 보유 → 패배.
5. 지훈이 안주 주문. 챌린지 종료.

## 시나리오 B — 1달 (월간 독박왕)
민수가 10월 1일 챌린지 제안.

1. 기간: 1달 / 게임: 구슬 레이스 / 벌칙: 꼴찌가 11월 술값 / 공개: 종료 후.
2. 10월 한 달간 매일 게임 알림 발송.
3. 만날 때 게임. 못 만나면 각자 앱으로 참여.
4. 미참여일은 자동 패배 처리.
5. 10월 31일 밤 12시 — 결과 공개 알림.
6. 독박왕: 민수. 결과 카드 단톡방 공유.
7. 11월 1일 — 새 챌린지 시작.

## 시나리오 C — 1등 베네핏 (술자리 n빵 면제)
현우, 민수, 지훈, 태양이 금요일 저녁 술자리에서 만났다.

1. 태양이 챌린지 제안.
2. 기간: 단판 / 게임: 구슬 레이스 / 베네핏: 1등은 오늘 술값 n빵 면제 / 공개: 실시간.
3. 제안 즉시 시작. 구슬 레이스 25초.
4. 현우 1등 → 오늘 술값에서 현우 몫 제외.
5. 나머지 셋이 n빵. 현우는 공짜 술.

벌칙이 아니라 베네핏이기 때문에 모두가 적극적으로 게임에 임한다. '내가 1등 하면 공짜'라는 동기.

## 시나리오 D — 1등 베네핏 (여행 중 방 배정 선택)
4명이 제주도 여행 중. 숙소에 방 배정을 결정해야 한다.

1. 민수가 챌린지 제안.
2. 기간: 단판 / 게임: 타이머 게임 / 베네핏: 1등이 오늘 방 배정 선택권 / 공개: 실시간.
3. 제안 즉시 시작. 타이머 게임. 0.01초 두 번 스톱.
4. 지훈 1등 → 좋은 방 선택.
5. 나머지는 남은 방 배정.

여행 중 소소한 선택권(방, 좌석, 메뉴)을 게임으로 결정. 갈등 없이 재미있게 해결된다.

# Design Principles

- **챌린지 생성은 60초 이내.**
- **결과는 드라마틱해야 한다.**
- **히든 랭킹으로 긴장감을 유지한다.**
- **시즌 공개는 이벤트여야 한다.**
- **기록은 영구 보존한다.**
- **미참여는 자동 패배. 단순하게 간다.**
