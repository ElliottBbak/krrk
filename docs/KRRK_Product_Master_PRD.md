# 핵심 포지셔닝
*KRRK는 사다리타기 앱이 아니다. 친구들 사이의 소소한 결정을 드라마로 만들고, 그 기록이 우리 그룹의 역사가 되는 웹 챌린지 플랫폼이다.*

커피 누가 사냐, 술값 누가 내냐, 방 배정 누가 고르냐 — 매번 가위바위보로 끝내던 그 결정들이 KRRK에서는 드라마가 된다.
그리고 그 드라마는 사라지지 않는다. 시즌 기록으로 쌓이고, 독박왕 연대기가 된다.

# 서비스명 & 톤 앤 매너
## KRRK — '긁'
'긁혔다', '내가 긁혔어' — 한국 인터넷 밈 '긁'에서 출발한 이름이다.

**의미**
- 벌칙에 걸린 사람 → 긁혔다
- 베네핏을 가져간 사람 → 긁었다
- 게임 자체 → 긁는 행위

**톤 앤 매너**

| 항목 | 방향 |
| --- | --- |
| 전체 감성 | 진지하지 않음. 밈 감성. 약간 B급 유머. |
| 언어 | 친구한테 말하듯. 격식 없음. 짧고 직접적. |
| UI | 과장된 리액션. 드라마틱한 결과 연출. |
| 결과 공유 | 캡처해서 단톡방에 올리고 싶어지는 디자인. |
| 에러 메시지 | 딱딱하지 않게. '긁혔다' 같은 언어 활용. |

# 플랫폼
## 웹 기반
앱 설치 없음. URL 하나로 바로 시작.

**방식 A — 링크 하나 공유**

| 단계 | 설명 |
| --- | --- |
| 1. URL 수신 | 그룹장이 단톡방에 그룹 URL 하나 공유. |
| 2. 게스트 입장 | 브라우저에서 바로 열림. 닉네임 입력 후 입장. |
| 3. 게임 참여 | 설치 없이 즉시 게임 참여. |
| 4. 개인 토큰 링크 발급 | 게임 후 개인 링크 발급. 카카오 나한테 보내기로 저장. |
| 5. 재입장 | 개인 토큰 링크로 접속 시 이전 참여자로 자동 인식. |
| 6. 영구 저장 (선택) | 소셜 로그인 연동 시 기록 영구 보존. 강제 아님. |

인원 미확정 시 사용. 누구나 들어올 수 있어야 할 때.

**방식 B — 개인 링크 생성**

| 단계 | 설명 |
| --- | --- |
| 1. 이름 등록 | 그룹 생성 시 참여자 이름 전원 미리 입력. |
| 2. 개인 링크 생성 | 참여자별 고유 URL 자동 생성. 전체 목록 단톡방에 공유. |
| 3. 개인 링크 클릭 | 각자 자기 이름 링크 클릭. |
| 4. 본인 확인 | '철수가 맞습니까?' 확인 후 즉시 입장. 닉네임 입력 없음. |
| 5. 잘못된 링크 | '이 링크는 영희용입니다' 안내. 올바른 링크로 재접속 유도. |
| 6. 재입장 | 개인 링크가 곧 개인 토큰. 저장해두면 다음에도 동일인으로 인식. |

인원 확정 시 사용. 닉네임 입력 없음. 가장 빠른 입장 방식.

*로그인 강제 없음. 방식 B는 개인 링크 자체가 식별자. 소셜 로그인은 영구 보존 원할 때만.*

## 인증 구조

| 방식 | 설명 |
| --- | --- |
| 게스트 | 닉네임만 입력. 개인 토큰 발급. 브라우저 세션에 저장. |
| 개인 토큰 링크 | 게임 후 발급. 저장해두면 며칠 후 재입장해도 같은 사람으로 인식. 링크 분실 시 재식별 불가. |
| 소셜 로그인 (선택) | 카카오 또는 구글. 기록 영구 보존 원할 때만. 게스트 기록 자동 병합. |

# 장기 해자
*경쟁사는 게임을 복제할 수 있다. 하지만 우리 그룹의 역사는 복제할 수 없다.*

**복제 가능**
- 게임 메커니즘
- UI/UX
- 랜덤 추첨 로직

**복제 불가능**
- 그룹 역사 및 타임라인
- 시즌 기록 및 독박왕 연대기
- 업적 데이터
- 3년간 쌓인 우리 그룹만의 전설

**기록이 해자다. 오래 쓸수록 다른 앱으로 이동할 이유가 없어진다.**

# 북극성 지표

***시즌 종료 후 같은 그룹이 다음 챌린지를 시작하는 비율***

이 숫자가 올라가면 KRRK는 성공하고 있다.
이 숫자가 낮으면 기록이 해자로 작동하지 않는 것이다.
