# 개발 원칙
*1개월 안에 친구들한테 보여줄 수 있는 것만 MVP. 나머지는 다 뒤로.*

- Phase 1 목표: 술자리에서 URL 던지고 구슬 레이스 한 판 돌리는 것.
- 완성도보다 핵심 경험 먼저. 디자인은 나중에 다듬는다.
- 각 Phase 끝에 실제로 써본다. 안 되면 고친다.
- Claude Code 활용. ERD + API Spec + 이 문서 던져주고 시작.

# 전체 타임라인

| Phase | 기간 | 핵심 목표 | 완료 기준 |
| --- | --- | --- | --- |
| Phase 1 | 1개월 (주말 4~5회) | 구슬 레이스 + 단판 | 친구들이랑 술자리에서 한 판 돌린다 |
| Phase 2 | 2개월 (주말 8~10회) | 폭탄 + 타이머 + 토큰 | 3가지 게임 전부 돌아간다 |
| Phase 3 | 3개월 (주말 12~15회) | 시즌제 + 히든 랭킹 | 월말 독박왕 공개가 작동한다 |
| Phase 4 | 이후 | 공유 + 히스토리 + 업적 | 결과 카드 단톡방 공유 된다 |

**Phase 1 ****— 구슬 레이스 + 단판**  1개월 · 주말 4~5회
*목표: 술자리에서 URL 던지고 구슬 레이스 한 판 돌린다. 그게 전부.*

**WEEK 1 — 프로젝트 셋업 + 인증**
○ NestJS + React + Vite + PostgreSQL + Socket.IO 초기 셋업
○ TypeORM 엔티티 생성 (users, groups, group_members, invite_links)
○ 게스트 입장 — 닉네임 입력 → 개인 토큰 발급
○ 방식 A: 링크 하나 공유 → 닉네임 입력 → 입장

**WEEK 2 — 그룹 생성 + 챌린지**
○ 그룹 생성 — 이름 입력 → 초대링크 생성 (24시간/7일)
○ 그룹 홈 — 활성 챌린지 목록
○ 챌린지 제안 — 단판 / 구슬 레이스 / 꼴찌 벌칙 / 공개 방식
○ 챌린지 제안 즉시 시작. 그룹 전원 자동 참여.

**WEEK 3 — 구슬 레이스 게임**
○ Socket.IO Gateway 셋업. 게임 룸 생성.
○ 구슬 레이스 엔진 — 멤버 수만큼 구슬 생성. 랜덤 낙하.
○ 구슬에 닉네임 표시. 실시간 위치 브로드캐스트.
○ 게임 종료 → 순위 계산 → game_results 저장.

**WEEK 4 — 결과 공개 + 마무리**
○ 결과 화면 — 꼴찌 확정. '긁혔다' 연출.
○ 단판 챌린지 종료. 기록 저장.
○ 기본 UI 다듬기. 모바일 웹 대응.
○ 친구들이랑 실제로 한 판 돌린다. 피드백 수집.

*Phase 1 완료 기준: 4명이 각자 폰으로 접속해서 구슬 레이스 한 판 완주.*

**Phase 2 ****— 게임 3종 + 토큰 재입장**  2개월 · 주말 8~10회
*목표: 3가지 게임 전부 돌아간다. 개인 토큰으로 재입장 가능.*

**폭탄 게임**
○ 폭탄 랜덤 패스 로직. 타이머 숨김.
○ 5초 전 — 흔들림 + 연기 연출. 보유자 화면 강조.
○ 폭발 애니메이션 + 꼴찌 확정.

**타이머 게임**
○ 순서 정하기 버튼 → 랜덤 순서 확정.
○ 순차적 진행 — 한 명씩. 전체 화면에 실시간 타이머 표시.
○ 0.01초 스톱 두 번. 마지막 자리 수 곱하기 계산.
○ 앞 사람 결과 공개 상태에서 내 차례 압박 연출.

**개인 토큰 링크**
○ 게임 후 개인 토큰 링크 발급.
○ 카카오 나한테 보내기 버튼.
○ 토큰 링크로 재입장 시 이전 참여자 자동 인식.

**방식 B — 개인 링크 생성**
○ 참여자 이름 등록 → 개인 URL 생성.
○ '철수가 맞습니까?' 본인 확인 후 입장.
○ 잘못된 링크 오류 처리.

*Phase 2 완료 기준: 3가지 게임 전환 가능. 재입장 시 기록 유지.*

**Phase 3 ****— 시즌제 + 히든 랭킹**  3개월 · 주말 12~15회
*목표: 1주/1달 챌린지. 히든 랭킹. 월말 독박왕 공개.*

**1주/1달 챌린지**
○ 1주/1달 기간 챌린지 생성.
○ 매일 게임 알림 발송.
○ 미참여 = 자동 패배 처리.
○ 동시 복수 챌린지 운영.

**히든 랭킹**
○ 종료 후 공개 모드 — 기간 중 순위 비공개.
○ 본인 상태만 표시 (안전권 / 위험권 / 매우 위험).

**시즌 종료 공개**
○ 기간 종료 → 결과 계산 → 전원 푸시 알림.
○ 드라마틱한 순위 공개 연출.
○ 독박왕 확정. season_summaries 저장.
○ 그룹 타임라인에 시즌 기록 누적.

**그룹 휴면**
○ 3개월 비활성 → 자동 휴면 전환 (Cron Job).
○ 휴면 탭 별도 표시. 복구 기능.

*Phase 3 완료 기준: 1달 챌린지 돌리고 월말에 독박왕이 공개된다.*

**Phase 4 ****— 공유 + 히스토리 + 업적**  이후
*목표: 결과가 단톡방에 퍼진다. 그룹의 역사가 쌓인다.*

**공유**
○ 결과 카드 이미지 생성. 단톡방 공유 버튼.
○ 카카오 소셜 로그인 연동.

**히스토리**
○ 그룹 타임라인 — 역대 시즌 기록 조회.
○ 멤버별 누적 기록 — 총 꼴찌 횟수, 1등 횟수.
○ 독박왕 연대기.

**업적**
○ 독박왕 (3연속 꼴찌), 연승왕 (5연속 1등), 역전왕 등.
○ 업적 획득 시 알림 + 연출.

**1등 베네핏 모드**
○ WINNER_BENEFIT 챌린지 전체 지원.
○ 1등왕 공개 연출.

*Phase 4 완료 기준: 결과 카드가 단톡방에 돌아다닌다.*
