KRRK

API Specification · NestJS REST + Socket.IO · v1.0 · 2025
Base URL: https://api.krrk.io · 모든 응답 형식: { data, message, statusCode } · 인증: Authorization: Bearer {token}
Auth API Phase 1
POST /auth/guest 게스트 입장 — 닉네임 입력 → 개인 토큰 발급
Request Body
nicknamestring필수최대 30자
inviteTokenstring선택초대링크 토큰. 있으면 그룹 자동 입장
Response
{ userId, accessToken, personalToken,
personalLink: "https://krrk.io/u/{token}" }
POST /auth/token 개인 토큰으로 재입장
Request Body
personalTokenstring필수카카오 저장해둔 개인 토큰
Response
{ userId, accessToken, nickname,
groups: [{ id, name, status }] }
POST /auth/social 카카오/구글 소셜 로그인 Phase 4
Request Body
providerstring필수KAKAO | GOOGLE
accessTokenstring필수소셜 액세스 토큰
mergeTokenstring선택게스트 기록 병합용 개인 토큰
Response
{ userId, accessToken, isNewUser }

Invite Links API Phase 1
POST /groups/:groupId/invites 초대링크 생성 (방식 A / B) auth
Request Body
typestring필수SHARED | PERSONAL
expiresInstring필수24h | 7d
membersstring[]선택방식 B: ["철수","영희","순자"]
Response
// 방식 A
{ url: "https://krrk.io/g/{token}",
expiresAt }

// 방식 B
{ links: [{ name: "철수",
url: "https://krrk.io/u/aaa1" },
...], expiresAt }
GET /invites/:token 초대링크 정보 조회 (입장 전 확인)
Response
{ groupId, groupName, type,
targetName?: "철수",
isExpired, memberCount }
비고
방식 B: targetName이 있으면 '철수가 맞습니까?' 확인 화면 표시. 만료 시 isExpired: true.
Challenges API Phase 1
POST /groups/:groupId/challenges 챌린지 제안 → 즉시 시작 auth
Request Body
durationstring필수SINGLE | WEEK | MONTH
typestring필수LOSER_PENALTY | WINNER_BENEFIT
gameTypestring필수MARBLE_RACE | BOMB | TIMER
rewardTextstring필수벌칙/베네핏 내용. 최대 200자
revealModestring필수REALTIME | ON_END
Response
{ id, groupId, duration, type,
gameType, rewardText,
revealMode, status: "ACTIVE",
startsAt, endsAt }
BOMB 게임은 WINNER_BENEFIT 챌린지에서 선택 불가. 서버에서 validation 처리.
GET /groups/:groupId/challenges 그룹 챌린지 목록 auth
Query Params
statusstring선택ACTIVE | FINISHED | REVEALED | ARCHIVED
Response
{ challenges: [Challenge] }
GET /challenges/:challengeId/my-status 히든 랭킹 — 내 상태만 조회 auth
Response
{ status: "SAFE" | "DANGER" | "CRITICAL",
gamesPlayed, gamesTotal,
// 구체적 순위/점수 없음 }

Response

Socket.IO 이벤트 상세 Phase 1
연결: wss://api.krrk.io · 룸: gameSession:{sessionId} · 인증: handshake.auth.token
Client → Server emit
join_game
{ sessionId: string, userId: string }
game_ready
{ sessionId: string } // 전원 ready 시 서버가 game_start 발송
timer_stop
{ sessionId: string, stopIndex: 1 | 2, value: number } // value: 0.01초 단위 타이머 값 (예: 3.47)
leave_game
{ sessionId: string }
Server → Client on
game_start
{ sessionId, gameType, players: [{ userId, nickname, avatarColor }], playerOrder?: string[] // 타이머 게임: 랜덤 순서 userId 배열 }
game_state
// 구슬 레이스: 매 프레임 브로드캐스트 { sessionId, marbles: [{ userId, x, y, rank }] } // 폭탄 게임: 폭탄 이동 시 { sessionId, bombHolder: userId, timeLeft?: null, warningMode: boolean } // 타이머 게임: 현재 차례 안내 { sessionId, currentTurn: userId, turnIndex: number }
timer_result
{ sessionId, userId, stopIndex: 1 | 2, value: number, // 3.47 lastDigit: number, // 7 score?: number // stopIndex=2일 때만: 7×2=14 }
game_end
{ sessionId, rankings: [{ rank, userId, nickname, score? }], loserId: string, // 꼴찌 winnerId: string, // 1등 rewardText: string }
season_reveal
{ challengeId, rankings: [{ rank, userId, nickname, wins, losses }], mvpUserId, mvpNickname, rewardText, cardImageUrl?: string // Phase 4 }
bomb_explode
{ sessionId, loserId: string } // 폭발 순간 별도 이벤트. 클라이언트에서 폭발 애니메이션 트리거.
에러 코드
코드 상황 메시지
400잘못된 요청gameType BOMB은 WINNER_BENEFIT에서 사용 불가
401인증 실패토큰 만료 또는 유효하지 않음
403권한 없음그룹 멤버가 아님
404초대링크 만료링크가 만료되었습니다. 새 링크를 요청해주세요.
409잘못된 개인링크이 링크는 영희용입니다. 철수 링크로 다시 입장해주세요.
423휴면 그룹휴면 상태입니다. 복구 후 이용 가능합니다.
