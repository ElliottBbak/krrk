테이블 관계
Group ↔ Member
groups1 ──── Ngroup_members한 그룹에 여러 멤버
users1 ──── Ngroup_members한 유저가 여러 그룹
Group ↔ Challenge
groups1 ──── Nchallenges동시에 여러 챌린지 가능
users1 ──── Nchallenges누구나 제안 가능
Challenge ↔ Game
challenges1 ──── Ngame_sessions챌린지 내 여러 판
game_sessions1 ──── Ngame_results판당 참여자별 결과
Challenge ↔ Season
challenges1 ──── 1season_summaries종료 후 생성
groups1 ──── Ninvite_links방식 A/B 모두
Socket.IO 이벤트 설계
Client → Server (emit)
join_game{ sessionId, userId } — 게임 룸 입장
game_ready{ sessionId } — 준비 완료
timer_stop{ sessionId, stopIndex, value } — 타이머 스톱
leave_game{ sessionId } — 게임 나가기
Server → Client (on)
game_start{ playerOrder, gameData } — 게임 시작. 순서 공개.
game_state{ state } — 실시간 게임 상태 브로드캐스트
timer_result{ userId, value, score } — 타이머 결과 전체 공개
game_end{ rankings, loserId, winnerId } — 게임 종료. 결과 공개.
season_reveal{ summary } — 시즌 종료 드라마틱 공개
NestJS 모듈 구조
src/
├── auth/             # 게스트·토큰·소셜 로그인. PersonalToken 발급.
├── users/            # User 엔티티. 토큰 재식별.
├── groups/           # Group CRUD. 휴면 자동 전환 (Cron). 초대링크 생성.
├── challenges/       # Challenge 제안·진행·종료. 히든랭킹 계산.
├── games/            # GameSession 생성. 게임별 로직 (Marble/Bomb/Timer).
├── gateway/          # Socket.IO Gateway. 실시간 이벤트 처리.
├── seasons/          # SeasonSummary 생성. 결과 카드 생성.
└── common/           # Enum. DTO. Guard. Interceptor.
